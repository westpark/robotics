Use zmq for queueing command

Inputs from:

* Local control (eg buttons on the Robot)
* Remote control (via zmq)
* Sensors

Commands to:

* Physical device
* Text
* Turtle-based graphical output

Programmes:

* Instantiate remote control
* Remote-control only
* Free Running
* Line-following
* Proximity
* etc.


Rename "robot" to "controller"
"input" is a zmq socket
"output" is an output object which will accept low-level commands, such as "stop-left", "forward-right"

Programme
- input from sensors
- input from remote socket
- instantiate low-level driver, specific to robot
